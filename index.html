🔍 Analisis Komparatif: Volatility 3 (Live Memory) vs Autopsy (Static Disk) dalam Investigasi Artefak Volatile dan Enkripsi Canggih

📌 Ringkasan Eksekutif: Artikel ini menyajikan analisis komparatif mendalam antara Volatility 3 (Live Memory Forensics) dan Autopsy (Static Disk Forensics) dalam konteks Digital Forensics and Incident Response (DFIR). Fokus utama adalah pada ekstraksi artefak volatile (RAM) dan enkripsi state-of-the-art (BitLocker XTS-AES, Android FBE, LUKS2), serta bagaimana keduanya saling melengkapi dalam Hybrid Forensic Workflow.

📚 1. Fondasi Teoretis: Hierarki Volatilitas & Keterbatasan Autopsy

1.1 Hierarki Volatilitas (RFC 3227)

Berdasarkan RFC 3227 (Guidelines for Evidence Collection and Archiving), data dalam RAM menempati urutan tertinggi dalam Order of Volatility:

  • Musnah seketika saat daya sistem diputus (power loss).
  • Tidak persisten (tidak tersimpan di disk).
  • Mengandung kunci enkripsi aktif, sesi jaringan, dan runtime state aplikasi.
Tingkat Volatilitas Jenis Data Contoh Artefak Tools yang Dibutuhkan
Tertinggi (1) RAM Kunci enkripsi, process environment, network sockets Volatility 3
Tinggi (2) Swap/Pagefile Data yang paged out dari RAM Volatility 3 + Autopsy
Sedang (3) Disk (Non-Volatile) File sistem, log, prefetch, registry Autopsy
Rendah (4) Backup/Archive Shadow copies, Time Machine Autopsy + FTK Imager

1.2 Keterbatasan Autopsy dalam Analisis Volatile

Autopsy bekerja dengan bit-stream imaging (E01, RAW/DD) dari non-volatile storage (HDD/SSD). Pendekatan ini tidak dapat:

  • ❌ Membaca kunci simetris dari aplikasi End-to-End Encryption (E2EE) yang berjalan di background.
  • ❌ Mendeteksi Process Environment Block (PEB) dari malware yang menyuntikkan kode ke memori proses legal (process hollowing).
  • ❌ Merekonstruksi runtime state (koneksi jaringan aktif, decryption keys sementara).
⚠️ Catatan Penting: Jika sistem menggunakan BitLocker, LUKS2, atau Android FBE, Autopsy hanya bisa menganalisis disk jika kunci pemulihan (recovery key) disediakan. Tanpa kunci, partisi terenkripsi akan terbaca sebagai unallocated space yang tidak dapat diindeks.

📊 2. Tabel Komparasi Fitur Teknis & Kapabilitas Ekstraksi

Dimensi Komparasi Autopsy (Static Disk Forensics) Volatility 3 (Live Memory Forensics)
Kondisi Target Perangkat Offline / Dead State (Sistem dimatikan) Online / Live State (RAM di-capture saat aktif)
Metode Akuisisi Bukti Bit-stream imaging (FTK Imager, Guymager, dd) Memory dumping (WinPmem, LiME, DumpIt)
Penanganan Enkripsi Disk Memerlukan recovery key / password eksternal Dapat mengekstrak master key langsung dari RAM

Integritas Bukti (Footprint) Tinggi (0% modifikasi pada media target) Ada footprint kecil di RAM saat proses dumping
Format Output Standar SQLite DB, Laporan HTML/PDF Teks terstruktur (JSON, CSV, Plain Text via plugin)
Ketergantungan Simbol Kernel Tidak ada Memerlukan PDB files (untuk Windows) / Debug Symbols (Linux)
Dukungan Sistem Operasi Windows, Linux, macOS (via file system parsing) Windows, Linux, macOS (tergantung profile)
Kemampuan Analisis Runtime ❌ Tidak ✅ Ya (process injection, memory artifacts)

🔧 3. Deep-Dive Metodologi Analisis Komparatif

3.1 Ekstraksi Kunci Enkripsi (Decryption Keys)

🔹 Menggunakan Autopsy

  • Keterbatasan:
    • Jika partisi terenkripsi (misal: BitLocker), Autopsy memerlukan kunci pemulihan (recovery key) secara manual.
    • Tanpa kunci, partisi akan terbaca sebagai unallocated space terenkripsi.
  • Cara Kerja: 
    # Contoh perintah di Autopsy (GUI-based)
# 1. Buka image disk (E01/RAW)
# 2. Pilih "Add Data Source" → "Disk Image"
# 3. Jika terenkripsi, Autopsy akan meminta kunci

🔹 Menggunakan Volatility 3

  • Keunggulan:
    • Ketika sistem berjalan, kunci BitLocker (FVEK - Full Volume Encryption Key) didekripsi ke RAM untuk operasi I/O.
    • Volatility 3 dapat mengekstrak FVEK langsung dari memory pool.
  • Cara Kerja (Kali Linux): 
    # 1. Capture RAM menggunakan LiME (Linux) atau WinPmem (Windows)
sudo lime -o lime_mem.dump

# 2. Ekstrak FVEK dari memory dump (Windows)
python3 vol -f lime_mem.dump windows.cachedump.Cache
python3 vol -f lime_mem.dump windows.bitlocker.BitLocker
📌 Catatan:
  • Untuk BitLocker, gunakan plugin windows.bitlocker.
  • Untuk LUKS2, gunakan linux.luks (jika tersedia) atau analisis manual dm-crypt keys di RAM.

3.2 Analisis Proses dan Eksekusi Kode

🔹 Menggunakan Autopsy

  • Keterbatasan:
    • Hanya bergantung pada artefak persisten (misal: Prefetch files .pf, Shimcache, UserAssist di Registry).
    • Tidak dapat mendeteksi reflective DLL injection atau process hollowing (karena tidak meninggalkan jejak di disk).

🔹 Menggunakan Volatility 3

  • Keunggulan:
    • Menganalisis struktur data kernel (misal: InLoadOrderModuleList, VAD - Virtual Address Descriptor).
    • Dapat mendeteksi halaman memori dengan hak akses PAGE_EXECUTE_READWRITE (ciri khas malware injection).
  • Cara Kerja (Kali Linux): 
    # 1. Scan untuk mendeteksi malware di memori
python3 vol -f lime_mem.dump windows.malfind

# 2. Analisis proses yang mencurigakan
python3 vol -f lime_mem.dump windows.pslist
python3 vol -f lime_mem.dump windows.pstree

# 3. Ekstrak DLL yang disuntikkan
python3 vol -f lime_mem.dump windows.dlllist --pid <PID>
python3 vol -f lime_mem.dump windows.ldrmodules --pid <PID>

# 4. Analisis VAD (Virtual Address Descriptor)
python3 vol -f lime_mem.dump windows.vadinfo --pid <PID>

3.3 Rekonstruksi Aktivitas Jaringan (Network State)

🔹 Menggunakan Autopsy

  • Keterbatasan: Hanya dapat menganalisis log historis (misal: Event Logs, Browser History, pcap tersimpan).

🔹 Menggunakan Volatility 3

  • Keunggulan: Dapat memetakan real-time network state (koneksi soket aktif, alamat IP eksternal, PID terkait).
  • Cara Kerja (Kali Linux): 
    # 1. Scan koneksi jaringan aktif
python3 vol -f lime_mem.dump windows.netscan

# 2. Analisis socket TCP/UDP
python3 vol -f lime_mem.dump windows.netstat

# 3. Ekstrak alamat IP dan port
python3 vol -f lime_mem.dump windows.sockets

⚖️ 4. Analisis Kelemahan Eksperimental (The Forensic Trade-Off)

Kriteria Autopsy Volatility 3 Keterangan
Risiko Kontaminasi Data ❌ Tidak ada ✅ Ada (footprint saat dumping) Prinsip Locard: RAM capture memodifikasi state memori.
Stabilitas Kernel ✅ Stabil ⚠️ Bergantung pada Symbol Files (PDB) Jika kernel custom (misal: Linux hardened), Volatility 3 mungkin gagal.
Dukungan Enkripsi ❌ Memerlukan kunci eksternal ✅ Dapat ekstraksi kunci dari RAM Hybrid Workflow disarankan.
Deteksi Fileless Malware ❌ Terbatas ✅ Efektif Volatility 3 unggul untuk memory-resident malware.
Kemudahan Penggunaan ✅ GUI (user-friendly) ⚠️ CLI (command-line) Autopsy lebih mudah untuk pemula.
💡 Kesimpulan: Tidak ada tools yang sempurna – keduanya saling melengkapi. Gunakan Volatility 3 untuk data volatile, lalu Autopsy untuk data persisten.

🛠️ 5. Panduan Praktis untuk Kali Linux

5.1 Instalasi Tools

🔹 Instalasi Autopsy

# 1. Tambahkan PPA Autopsy (jika diperlukan)
sudo add-apt-repository ppa:sleuthkit/autopsy
sudo apt update

# 2. Instal Autopsy
sudo apt install autopsy -y

# 3. Jalankan Autopsy (via browser)
autopsy
# Buka http://localhost:9999 di browser

🔹 Instalasi Volatility 3

# 1. Clone repository Volatility 3
git clone https://github.com/volatilityfoundation/volatility3.git
cd volatility3

# 2. Buat virtual environment (disarankan)
python3 -m venv venv
source venv/bin/activate

# 3. Instal dependencies
pip install -r requirements.txt

# 4. Verifikasi instalasi
python3 vol --info

🔹 Instalasi Tools Tambahan (Memory Dumping)

# 1. LiME (Linux Memory Extractor)
git clone https://github.com/504ensicsLabs/LiME.git
cd LiME/src
make
sudo cp lime.ko /lib/modules/$(uname -r)/kernel/

# 2. WinPmem (untuk Windows, jalankan di target)
# Download dari: https://github.com/Velocidex/caffeine
# Contoh penggunaan:
# .\WinPmem_mini_x64.exe -o memory.dump

5.2 Workflow Hybrid Forensics (Kali Linux)

📌 Langkah 1: Akuisisi Memory (Live Forensics)

# 1. Capture RAM menggunakan LiME (Linux)
sudo lime -o /tmp/memory.dump

# 2. Atau gunakan AVML (jika target Windows)
# Download AVML: https://github.com/microsoft/avml
# avml.exe -o memory.avml

📌 Langkah 2: Analisis Memory dengan Volatility 3

# 1. Identifikasi profil sistem (Windows/Linux)
python3 vol -f /tmp/memory.dump windows.info
# Catat profil (misal: Win10x64_19041)

# 2. Ekstrak kunci enkripsi (BitLocker)
python3 vol -f /tmp/memory.dump --profile Win10x64_19041 windows.bitlocker.BitLocker

# 3. Deteksi malware
python3 vol -f /tmp/memory.dump --profile Win10x64_19041 windows.malfind

# 4. Analisis jaringan
python3 vol -f /tmp/memory.dump --profile Win10x64_19041 windows.netscan

📌 Langkah 3: Akuisisi Disk (Static Forensics)

# 1. Buat image disk menggunakan dd (hati-hati: lambat untuk disk besar)
sudo dd if=/dev/sdX of=/tmp/disk.img bs=4M status=progress

# 2. Atau gunakan Guymager (GUI)
sudo apt install guymager
guymager

📌 Langkah 4: Analisis Disk dengan Autopsy

# 1. Buka Autopsy di browser
autopsy

# 2. Tambahkan image disk (E01/RAW)
# 3. Gunakan kunci enkripsi yang diekstrak dari RAM (jika diperlukan)
# 4. Analisis artefak persisten (Registry, Logs, dll.)

📌 Langkah 5: Integrasi Hasil

  • Gunakan kunci enkripsi dari Volatility 3 untuk membuka partisi terenkripsi di Autopsy.
  • Bandinkan temuan dari kedua tools untuk rekonstruksi lengkap insiden.

5.3 Tips & Trik untuk Kali Linux

  • ✅ Gunakan screen atau tmux untuk sesi memory dumping yang panjang: 
    screen -S forensics
sudo lime -o /tmp/memory.dump
# Tekan Ctrl+A, lalu D untuk detach
  • ✅ Simpan symbol files (PDB) untuk Windows: 
    # Download PDB dari Microsoft Symbol Server
# Contoh untuk Windows 10 19041:
wget https://msdl.microsoft.com/download/symbols/Windows10_19041.pdb
  • ✅ Gunakan volatility3 dengan plugin tambahan: 
    # Clone plugin tambahan (misal: untuk Linux)
git clone https://github.com/volatilityfoundation/volatility3-plugins.git
  • ✅ Analisis timeline dengan timesketch (integrasi Autopsy): 
    sudo apt install timesketch

📊 Alur Kerja Hybrid Forensics

Start → Live Memory Capture → Analisis Memory (Volatility 3) → Ekstrak Kunci & Artefak Volatile
→ Matikan Sistem → Disk Imaging → Analisis Disk (Autopsy) → Integrasi Hasil → Laporan Forensik

🎯 6. Kesimpulan & Rekomendasi

6.1 Tesis Utama

Tools Kelebihan Kekurangan Kapan Digunakan?
Autopsy ✅ Stabil, GUI, user-friendly ❌ Tidak bisa analisis volatile data Static Forensics (disk, file sistem, log)
Volatility 3 ✅ Dapat analisis RAM, runtime state, encryption keys ⚠️ Memerlukan symbol files, CLI Live Forensics (memori, fileless malware, network state)
🔥 Rekomendasi: Gunakan keduanya dalam Hybrid Workflow.
  1. Lakukan Live Memory Forensics terlebih dahulu (Volatility 3) untuk mengamankan:
    • Decryption keys (BitLocker, LUKS2, FBE).
    • Network connections aktif.
    • Fileless malware di memori.
  2. Matikan sistem secara aman (orderly shutdown).
  3. Lakukan Disk Imaging (Autopsy) untuk analisis persistent artifacts.

🔐 8. Catatan Hukum & Chain of Custody

8.1 Prinsip Chain of Custody

  • Dokumentasikan setiap langkah (tanggal, waktu, tools, hash).
  • Gunakan hashing (SHA-256) untuk memverifikasi integritas bukti: 
    sha256sum /tmp/memory.dump > memory_dump_hash.txt
sha256sum /tmp/disk.img > disk_image_hash.txt
  • Simpan log semua perintah yang dieksekusi: 
    script forensics_log.txt
# Jalankan semua perintah di sini
exit

8.2 Standar Hukum (Indonesia)

  • UU ITE No. 11 Tahun 2008 (Pasal 5: Bukti elektronik sah sebagai bukti hukum).
  • Peraturan Kapolri No. 10 Tahun 2019 (Tata cara penanganan bukti elektronik).
  • ISO/IEC 27037 (Guidelines for identification, collection, acquisition, and preservation of digital evidence).
⚠️ Peringatan:
  • Jangan modifikasi bukti asli (selalu kerja pada copy).
  • Gunakan write blocker saat melakukan disk imaging (misal: dd dengan conv=noerror,sync).
  • Dapatkan izin hukum sebelum melakukan forensic analysis pada sistem yang bukan milik Anda.
